# 🧠 MeetMind — AI Meeting Intelligence Agent

> Drop in a YouTube link or local video — get a full transcript, smart summary, action items, key decisions, open questions, and a live RAG-powered chat interface.

---

## 📌 What is MeetMind?

MeetMind is an **end-to-end AI pipeline** that transforms any meeting recording into structured, actionable intelligence.

**Input:** YouTube URL or local video/audio file
**Output:**

| Output | Description |
|--------|-------------|
| 📄 Full Transcript | Complete speech-to-text of the meeting |
| 🏷️ Auto Title | Professional 8-word title generated by AI |
| 📝 Summary | Bullet-point summary using Map-Reduce |
| ✅ Action Items | Tasks with owners and deadlines |
| 🔑 Key Decisions | All important decisions made |
| ❓ Open Questions | Unresolved follow-up questions |
| 💬 RAG Chat | Ask anything — answers grounded in the transcript |

## ⚙️ How the Pipeline Works

The pipeline runs in **6 sequential steps** every time you analyse a meeting:

```
User Input (YouTube URL or local file)
        │
        ▼
[Step 1] Audio Processing    → Download, convert, split into 10-min chunks
        │
        ▼
[Step 2] Transcription       → Whisper (English) or Sarvam AI (Hindi/Hinglish)
        │
        ▼
[Step 3] Summarization       → Map-Reduce via Mistral AI → Title + Summary
        │
        ▼
[Step 4] Extraction          → Action Items, Key Decisions, Open Questions
        │
        ▼
[Step 5] Vector Store        → Embed transcript → ChromaDB (local)
        │
        ▼
[Step 6] RAG Chain           → LangChain LCEL + Mistral AI → live chat
        │
        ▼
Flask REST API → React Frontend
```

---

### 🎧 Step 1 — Audio Processing (`utils/audio_processor.py`)

Handles two input types:

**YouTube URL:**
```
yt-dlp downloads audio (best quality)
  → FFmpeg converts to .wav
  → Saved to: downloads/<video-title>.wav
```

**Local File (MP4 / MP3 / WAV):**
```
pydub loads the file
  → Convert to mono, 16 kHz .wav  (required by Whisper)
  → Saved as: <filename>_converted.wav
```

**Both paths then go through audio chunking:**
```python
chunk_size = 10 minutes (600,000 ms)
Output: [file_chunk_0.wav, file_chunk_1.wav, ...]
```

> **Why chunk at 10 minutes?**
> Whisper and Sarvam AI both have file size and memory limits. Chunking keeps processing stable for long recordings (1–3 hour meetings).

---

### ✍️ Step 2 — Transcription (`core/transcriber.py`)

Routes each audio chunk to the correct engine based on language choice:

**English / Auto → OpenAI Whisper (runs locally)**
```python
model = whisper.load_model("small")     # loaded once, cached
result = model.transcribe(chunk_path)
# Returns: plain English text string
```
- Runs 100% on your machine — no internet needed for transcription
- Configurable model size via `.env` (tiny → large)

**Hindi / Hinglish → Sarvam AI (cloud API)**
```
Sarvam API limit = max 30 seconds per request

So each 10-min chunk is split AGAIN into 25-second pieces:
  piece_0.wav → POST https://api.sarvam.ai/speech-to-text-translate
  piece_1.wav → POST ...
  piece_N.wav → POST ...
  All responses joined → full chunk transcript

Temp piece files are deleted after upload.
```

> **Double Chunking for Hinglish:**
> `audio_processor` → 10-min audio chunks
> `transcriber` → 25-sec Sarvam pieces
> This two-level chunking handles Sarvam's 30s API limit automatically.

All chunk transcripts are joined into one **full_transcript string**.

---

### 📝 Step 3 — Summarization (`core/summrizer.py`)

**Model:** `mistral-small-latest` | **Temperature:** `0.3`

#### Title Generation
```
Input: first 2000 characters of transcript
Prompt: "Generate a title no longer than 8 words"
Output: e.g. "Q3 Product Roadmap Strategy Planning Call"
```

#### Summary — Map-Reduce Pattern
```
SPLIT:
  RecursiveCharacterTextSplitter(chunk_size=3000, chunk_overlap=200)
  → list of text chunks

MAP:
  Each chunk → Mistral AI → bullet-point summary of that chunk

REDUCE:
  All chunk summaries combined → Mistral AI → single final summary
```

> **Why Map-Reduce?**
> Long meeting transcripts exceed the LLM's context window. Map-Reduce processes them in parts and merges intelligently — giving accurate summaries for 1-hour+ meetings.

---

### 🔍 Step 4 — Information Extraction (`core/extractor.py`)

**Model:** `mistral-small-latest` | **Temperature:** `0.2` (low = more factual)

Three separate LangChain LCEL chains run on the full transcript:

| Function | Prompt Goal | Output |
|----------|-------------|--------|
| `extract_action_items()` | Tasks + owner + deadline | Numbered list |
| `extract_key_decisions()` | All decisions made | Numbered list |
| `extract_questions()` | Unresolved follow-up questions | Numbered list |

> **Why temperature=0.2?**
> Lower temperature = more deterministic output. The LLM is less likely to hallucinate tasks or decisions that were never actually discussed.

---

### 🗄️ Step 5 — Vector Store (`core/vector_store.py`)

**Embedding Model:** `all-MiniLM-L6-v2` (HuggingFace, runs locally on CPU)
**Database:** ChromaDB (persisted to `./vector_DB/`)

```
Transcript text
  → RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
  → list of small text chunks
  → HuggingFace embeddings → 384-dimensional vectors
  → Stored in ChromaDB collection: "meeting_transcript"

Retriever: similarity search, returns top 4 matching chunks
```

#### All Chunking Levels at a Glance

| Level | File | Size | Why |
|-------|------|------|-----|
| Audio chunks | `audio_processor.py` | 10 min | Whisper/Sarvam file limits |
| Sarvam pieces | `transcriber.py` | 25 sec | Sarvam API 30s limit |
| Summary chunks | `summrizer.py` | 3000 chars (200 overlap) | LLM context window |
| RAG chunks | `vector_store.py` | 500 chars (50 overlap) | Precise semantic search |

---

### 💬 Step 6 — RAG Engine (`core/rag_engine.py`)

**Model:** `mistral-small-latest` | **Temperature:** `0.2`

```
User Question
  → ChromaDB retriever → top 4 relevant transcript chunks
  → format_docs() → chunks joined as context
  → Mistral AI prompt:
      System: "Answer ONLY from the meeting transcript context below.
               If not found, say: 'I could not find this information.'"
      Context: [4 relevant chunks]
      Human: [user question]
  → Grounded answer returned
```

Built using **LangChain LCEL** (`|` pipe syntax):
```python
rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}
    | prompt | llm | StrOutputParser()
)
```

### 🌐 Flask REST API (`api_server.py`)

Exposes the pipeline as async HTTP endpoints for the React frontend.

| Endpoint | Method | Body | Description |
|----------|--------|------|-------------|
| `/api/health` | GET | — | Check server + pipeline status |
| `/api/analyse` | POST | `{ source, language }` | Start analysis job |
| `/api/status/<job_id>` | GET | — | Poll progress (0–100%) + get result |
| `/api/chat/<job_id>` | POST | `{ question }` | Ask RAG question |

**Async Job Flow:**
```
POST /api/analyse
  → Creates UUID job_id
  → Starts background thread (run_pipeline)
  → Returns { job_id } immediately (202 Accepted)

Frontend polls: GET /api/status/<job_id>
  → { status, progress, log, result? }

When status = "done":
  result = { title, transcript, summary, action_items, decisions, questions }
```

> **Why async?** The pipeline takes 2–10 minutes. Async jobs prevent HTTP timeouts and keep the UI responsive with live progress updates.

### ⚛️ React Frontend (`meetmind-ui/`)

Built with **React 19 + Vite + Axios + React Router**.

```
User enters YouTube URL / file path + picks language
  → POST /api/analyse → receives job_id
  → Polls /api/status/<job_id> every few seconds
  → Shows live progress bar + step-by-step status logs
  → On completion, renders results:

      📋 Summary Tab
      ✅ Action Items Tab
      🔑 Key Decisions Tab
      ❓ Open Questions Tab
      📄 Full Transcript Tab

  → RAG Chat:
      User types question
      → POST /api/chat/<job_id>
      → Displays grounded answer
```

## 🚀 Setup & Run

### 1. Clone the Repository

```bash
git clone <your-repo-url>
cd "GenAi project"
```

---

### 2. Backend Setup (Python + Flask)

```bash
# Navigate to backend
cd "video agent "

# Create virtual environment
python -m venv videoVenv

# Activate virtual environment
source videoVenv/bin/activate          # macOS / Linux
# videoVenv\Scripts\activate           # Windows

# Install dependencies
pip install -r requirements.txt
```

#### Create `.env` file

```bash
# Create .env in the "video agent" folder
touch .env
```

Add your API keys:

```env
MISTRAL_API_KEY=your_mistral_api_key_here
SARVAM_API_KEY=your_sarvam_api_key_here
WHISPER_MODEL=small
SARVAM_MODEL=saaras:v2.5
```

**Where to get API keys:**
- **Mistral AI** → [https://console.mistral.ai](https://console.mistral.ai) (free tier available)
- **Sarvam AI** → [https://www.sarvam.ai](https://www.sarvam.ai) (only needed for Hindi/Hinglish)

#### Start the Backend

```bash
python api_server.py
# ✅ Running at http://localhost:8000
```

## 🛠️ Troubleshooting

| Problem | Solution |
|---------|----------|
| `SARVAM_API_KEY is not set` | Add `SARVAM_API_KEY=...` to your `.env` file |
| `ffmpeg not found` | Run `brew install ffmpeg` (macOS) |
| Whisper model download slow | First run downloads the model — this is normal, wait for it |
| `ModuleNotFoundError` | Make sure virtualenv is active and `pip install -r requirements.txt` was run |
| Old ChromaDB data showing | Delete `./vector_DB/` folder and rerun |
| CORS error from React | Ensure `flask-cors` is installed and `CORS(app)` exists in `api_server.py` |
| Frontend can't reach backend | Confirm backend is running on port 8000 |
